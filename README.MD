## Overview

### Challenge

Users should be able to:

- View the optimal layout depending on their device's screen size (mobile and desktop)
- View the recipe card with proper spacing, typography, and border-radius
- See preparation time, ingredients, instructions, and nutrition table
- Navigate through the recipe sections with proper heading hierarchy

## My Process

### Built With

- Semantic HTML5
- CSS (Flexbox / Responsive Design)
- Desktop-first workflow
- CSS custom properties (variables)
- Google Fonts (Young Serif and Outfit)

---

### What I Learned

- Creating a responsive recipe card layout with proper spacing and typography
- Using CSS custom properties in `:root` for maintainable colors and fonts
- Implementing `::marker` pseudo-element to style list bullets and numbers
- Building a nutrition table with proper borders and spacing
- Using `min-height: 100vh` to ensure background covers full screen
- Following a `style-guide.md` to ensure design fidelity

---

### Continued Development

- Add `:focus-visible` for better keyboard accessibility
- Refactor CSS for better maintainability
- Experiment with CSS Grid for future layouts
- Improve accessibility with ARIA attributes
